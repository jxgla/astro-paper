在这个 AI 浪潮奔涌的时代，作为一名开发者，拥有一款真正属于自己、甚至全方位掌控的 AI 助手是一件极具诱惑力的事情。

今天，我将复盘这一整天高强度的“疯狂”开发历程。我们没有使用传统的 XML 或笨重的跨平台框架，而是纯粹基于 **Kotlin + Jetpack Compose** 构建出了一款极具未来感与丝滑质感的 AI 原生客户端——命名为 **Aura**。

这是一场从“基础积木搭设”到“云端全栈掌控”，再到最终“赛博丝滑重构”的技术马拉松。以下是全栈开发的干货复盘。

## 阶段一：奠定基石（底层架构与通信生命线）

任何庞大的工程都需要坚实的底座。Aura 的定位不是一个套壳网页，而是一个高性能的 Android 原生应用。

1. **技术栈选型与项目骨架**
   - **UI 引擎**：放弃 XML，全面拥抱 **Jetpack Compose**，用声明式 UI 换取卓越的动态交互能力与开发效率。
   - **本地存储**：接入 **Room Database** 构建 `ChatDao`。我们需要一个极快、极稳的本地知识库来充当大模型的“长期记忆”。
   - **网络中枢**：引入 **Retrofit + OkHttp**。构建了严密的网络请求拦截器，无论是基础的请求头伪装，还是动态获取的 Token 注入，都在底层优雅拦截并完成。

2. **跨越网络长城：Cloudflare Worker Backend**
   - 为了实现最快、最安全的接口流转，并完美规避直连限制报错，我们在后端采用了 **Cloudflare Worker** 进行代理和逻辑包装。
   - 我们在这里处理了诸如 CORS 跨域、Options 预检请求慢等经典前端痛点，确保了 App 发出的每一个请求都能毫秒级穿透并获得响应。

## 阶段二：重塑核心体验（Gemini 风格与智能化进阶）

有了能说话聊天的基本功能后，我们开始追求“好用”。普通的聊天框太平庸，我们要的是充满掌控感的设计。

- **Gemini 风格的极简布局**
   我们引入了 Drawer 侧边抽屉、纯净的深色系与悬浮式胶囊输入框（Pill Input）。在胶囊内部优雅地融合了语音输入和图片附件的图标，做到了像素级的极简克制。
   
- **Zero-Barrier Onboarding (零门槛新手引导)**
   考虑到第一印象，我们开发了 `OnboardingManager`。对于带着激活码首次进入的用户，我们没有冷冰冰的登录框，而是一个经过精心设计的欢迎弹窗动画，优雅地接过设备的控制权。

- **Silent Auto-Cleanup (静默清理管家)**
   手机空间很宝贵，我们在 `SplashViewModel` 中埋下了一套自动清理逻辑。每一次 App 启动，系统都会在千分之一秒内查杀 7 天前未被标记为“星标收藏”的旧历史记录，而这一切用户毫无察觉。

## 阶段三：视觉与多模态的终极跃迁

AI 已经不是单纯的语言模型了，Aura 必须拥有“看”和“展示”的能力。

1. **OpenAI Vision 格式协议融合**
   - 传统 String 不能塞下图片。我们连根拔起重构了 `ApiModels.kt`，将请求体改为符合协议的 `List<ChatMessagePart>`。
   - 在图文混合发送时，由 Compose 在客户端完成高清图片选取、本地 BitmapFactory 端侧像素级压缩（规避 OOM 并缩减网传体积），再转换为 Base64 直传后台。

2. **Markdown 与网络图文流式渲染**
   - 有时候大模型会给你返回带配图的方案。为了破解“纯文本死链”的魔咒，我们在 `MessageBubble` 中内植入了强大的正则匹配树与 **Coil** 流式图片加载引擎。
   - 现在，只要 AI 返回 `![描述](图片URL)`，Aura 都会瞬间排版出精美的图文混排磁贴。为了防止 AI 产生幻觉（例如把上一轮自己画的图当成是用户发送的），我们更在传网上游做了一道“洗数据”的视觉错觉滤镜。

## 阶段四：赋予灵魂（自动重命名与极客掌控）

随着聊天增多，满屏的“新的对话”显然是不合格的体验。

- **AI 赋能 UI：语义截取重命名**
  当一个新会话聊出第一句话时，应用会在底层截取头 10 个字符作为语义种子，在后台线程自动更新 Session 名称，并伴随自定义的**视差滚动条**刷新侧边抽屉。这种自理能力，让它仿佛活着一般。

- **生命周期掌控与云端上帝视角**
  我们在终端加入了严格的隐私提示法务条款。而对于作为开发者的我们，依靠背后的 **Cloudflare D1 数据库**，只需一句 SQL 指令即可掌控全局：
  - 封禁攻击设备 (`UPDATE devices SET status = 'banned'`)
  - 动态切换特定用户的 NSFW 限制口径
  真正的做到了数据游刃有余。

## 最终章：名为 Aura 的科幻新生

到了当天开发的最后半小时，我们将这个名为“智能助手”的毛坯房，彻底装修成了高奢旗舰机预装套件——**Aura**。

1. **Adaptive Icon 桌面自适应雕琢**
   - 彻底摒弃了原来会被强行拉伸的边缘留白图片。我们动用了 Android 原生的**分层图层架构 (Adaptive Icon)**。
   - 采用了极为静谧的深海黑 (`#0A0A0C`) 作为底衬，Aura 原型的变色标识只占正中央。这样一来，不论是折叠屏、刘海屏甚至是曲模圆角，它都严丝合缝。

2. **Compose Animatable：呼吸的艺术启动**
   - 打开应用，不再是枯燥的圆形转圈。
   - 你将直面深邃放射状渐变（Radial Gradient）的浩瀚黑色。红紫渐变的 “A” 字如心脏般伴随阻尼弹簧动画 (`Spring`) 跃然放大。
   - 下方是无衬线字体优雅浮现的使命宣言：“*探索智能的边界*”。
   - 最终，这一切化作一阵持续数秒的光晕褪散 (`Crossfade`)，丝滑地带你推开那扇白色聊天厅的大门。

就这样，在不到一天的时间里，依靠着 Kotlin 与 Jetpack Compose 强大的表现力，一款集成了端侧数据、本地模型记忆、多模态智能以及电影级 UI 体验的个人应用拔地而起。

它的成长远未结束，但它的基底已经如同它的名字“Aura”一般，开始绽放光芒。
